求职者面试相关讨论
会议讨论了求职者的简历情况、技术能力、面试技巧等方面的问题，并给出了相应的学习规划和面试建议，具体如下：
- **个人经历与技术栈**：
  - **工作经历**：求职者陈远毕业在盛世科技工作四年，2月底因裁员离职，技术栈主要是Vue3、Vue2和少量Node.js，负责公司管理后台及官网维护。
  - **项目经验**：做过智能协作平台项目，涉及AI智能问答模块和流式处理；进行过Vue2到Vue3迁移；搭建过Sentry前端监控体系；建设过企业级组件库；有前端性能优化相关经验，但部分项目细节需完善。
- **项目问题分析与建议**：
  - **Sentry搭建**：目前只是简单调用API并存储JSON，建议自部署Sentry，通过Docker在本机安装，了解各服务功能，结合项目测试并分析。
  - **Vue2到Vue3迁移**：迁移中遇到写法适应问题和Webpack启动慢问题，需深入理解原理，明确第一次启动慢的原因，避免被面试官问住。
  - **Webpack升级**：从Webpack4升级到Vite，面试官可能会问为何不选Webpack5，需考虑各版本优缺点，提前调研迁移方案。
  - **组件库建设**：基于Element Plus二次封装组件库，需了解样式覆盖方法，考虑版本更新同步和业务方覆盖问题。
  - **前端性能优化**：了解一些常规优化方法，如CDN、懒加载等，但需学习Webpack分包、缓存（强缓存和协商缓存）等知识。
- **基础技术问题**：
  - **Vue基础**：回答Vue中v-if和v-show区别、Vue2和Vue3双向数据绑定原理等问题时较含糊，需夯实Vue基础，理解ref和reactive区别及实现原理。
- **面试技巧与建议**：
  - **回答问题方式**：面试官问什么答什么，不过度发散，除非特别有把握。
  - **自我介绍**：要丰富有亮点，吸引面试官提问。
  - **面试流程**：普通公司一般三面，一面小组长，二面总监，一面面试官需提供足够亮点推荐到二面，二面基于一面结果确认候选人特质。
- **学习规划与目标**：
  - **短期目标**：先涨工资，调整简历，突出业务能力，夯实Vue基础，学习工程化课程。
  - **长期目标**：冲击大厂，需深入学习Vue底层原理，掌握大厂高频手写题，完善项目经历。
- **其他注意事项**：
  - **离职原因**：面试时若被问离职原因，可说是部门调整或如实说裁员，工作四年多被裁没问题，6个月内被裁可能影响印象。
  - **模拟面试**：面试前进行模拟面试，避免拿大厂练手，大厂有面评影响后续面试。
  - **背调**：近5年简历无虚假信息，背调关注入职时间真实性和沟通协作等软技能。
  - **开源贡献**：给开源项目提PR要有一定深度和数量，避免提简单问题。
- **任务**
  - **Sentry 部署**：安装 Docker，需使用 VPN；按网上 Centos 自部署文档在本机安装 Centos，列出 Docker 服务，用 AI 解释服务功能，接入项目测试并分析流程。 
  - **Sentry部署总结**：从网上查找 sentry 自部署材料，安装 Docker 和 Centos 系统，列出服务用 AI 分析，接入 sentry SDK 触发报错，总结自部署运维能力并写进简历 
  - **工具试用对比**：亲自试用 WEBPACK5，对比 Vite 与 WEBPACK5 的构建速度，为面试充分阐述迁移到 Vite 或升级到 WEBPACK5 的原因做准备 
  - **组件库设计要点了解**：通过询问 AI 列出设计企业级设计组件库需注意的点，对每个点进行初步分析，避免面试时被问破防。 
  - **性能优化学习**：学习前端性能优化常规实践，包括网络优化、图片懒加载、Webpack 分包等，可通过课程学习工程化内容，了解 Webpack 多入口分包及利用 HTTP 头标识的强缓存和协商缓存。 
  - **REF知识回顾**：重新回顾 `REF` 和 `reactive` 的区别，了解 `REF` 实现方式，掌握 `REF` 能同时兼容基础类型和引用类型的原因，以便向面试官清晰解释 
  - **劫持实现查看**：查看非对象情况下 REF 劫持的实现方式，了解 REF 是如何通过 new 一个 class 实现 get 和 set 方法进行劫持的。 
  - **简历准备**：准备两份简历，一份用于涨工资，减少公众号内容、增加做业务内容；一份用于面大厂，按现有内容准备，同时加强 view 底层学习、背手写题、深挖项目。 
  - **简历内容修改**：修改简历，将 Webpack 迁到 Vite 的内容改成 WEBPACK5，保留前端优化和 centry 的搭建，组件库先引用，view2 升 view3 先 hold 一下
  - **VIEW 问题思考**：借助 AI 工具，思考 VIEW3 为何如此设计，以及 VIEW3 升级后解决了 VIEW2 的哪些问题，同时确保源码知识准确，避免回答错误。 
  - **简历问题完善**：拿着改过的简历，让 AI 提问或自行提问并回答，答不上来则找资料完善，直至无明显问题，面试前可约模拟面评估。 
  - **学习计划制定**：根据说话人 2 情况制定学习计划，列出需补充课程，发给班主任，由班主任转发给说话人 2
